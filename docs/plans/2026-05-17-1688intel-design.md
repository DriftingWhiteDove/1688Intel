# 1688Intel — 1688 电商情报工作站 设计文档

## 概述

面向淘宝、拼多多、抖音卖家的 1688 选品与供应商背调工具。帮助买家快速判断"这个品值不值得做"、"这家供应商靠不靠谱"。

## 核心功能

### 第一阶段（MVP）

| 模块 | 功能 | 输入 | 输出 |
|------|------|------|------|
| 商品分析 | AI 选品评分、利润估算 | 1688 商品链接 | 评分报告（推荐度/利润/风险） |
| 供应商背调 | 工商信息、刷单检测 | 店铺链接/ID | 背调报告（资质/信用/风险） |

### 第二阶段

- 批量导入/批量分析
- 关注商品价格变动追踪
- 小程序上线

### 第三阶段

- 跨平台比价（1688 vs 淘宝/拼多多）
- 品类趋势热力图

## 技术栈

| 层 | 技术 | 选型理由 |
|----|------|---------|
| 后端框架 | Python FastAPI | 异步支持好，自动生成 OpenAPI 文档 |
| 数据抓取 | Playwright（Python） | 模拟浏览器，反爬强，能跑 JS |
| 内部 API 请求 | httpx + sign 签名 | 直接调 1688 内部接口，效率高 |
| AI 分析 | DeepSeek API | 性价比高，中文电商理解好 |
| 数据库 | SQLite | 本地部署无需额外服务 |
| 前端 | Vue3 + Element Plus | 成熟稳定，适合管理后台 |
| 小程序 | 微信原生 / uniapp | MVP 之后再接入 |
| 任务调度 | APScheduler | 定时扫描/追踪任务 |

## 数据流

### 商品分析流程

```
用户输入 1688 商品 URL
  → Playwright 打开商品页
    → 拦截内部 API 获取：价格、销量、评价数、评分
    → 提取商品标题、主图、SKU
  → 数据传给 AI 分析引擎
    → DeepSeek 输出：推荐度评分 + 理由 + 风险提示
  → 利润计算器
    → 根据售价、类目费率估算利润空间
  → 输出结构化报告
```

### 供应商背调流程

```
用户输入店铺链接
  → 提取 memberId
  → 调 1688 内部 API（sign 签名）
    → 获取工商信息（公司名、注册资本、成立时间）
  → Playwright 获取店铺评价数据
    → AI 分析评价真实性（时间分布、内容重复度）
  → 输出背调报告
```

## 数据模型

```python
# 商品分析报告
ProductReport:
  url: str
  title: str
  price_range: [float, float]
  monthly_sales: int
  review_count: int
  avg_rating: float
  ai_score: float          # AI 推荐度 0-100
  ai_summary: str          # AI 分析摘要
  profit_estimate: dict    # 利润估算
  risk_flags: list[str]    # 风险标记

# 供应商背调报告

  shop_name: str
  company_name: str
  registration_info: dict  # 工商信息
  years_in_business: int
  trust_score: float       # 信用评分
  fake_review_warning: str # 刷单检测结论
  ai_assessment: str       # AI 综合评估
```

## 部署策略

- **本地模式**：用户在自己电脑跑后端 → 浏览器打开 localhost Web UI
- **小程序模式**：后端部署到轻量云服务器 → 微信小程序 HTTPS 连接
- 同一套代码，两套部署方式
